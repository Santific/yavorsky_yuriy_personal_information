Мои приложения под Android

В папке Android находятся скрины/apk из приложений!

1) Приложение для отображение погоды городов. (С исспользованием Service, ContentProvicder, AlarmManager)
	Поиск города на сервере.
	Отображение погоды в городе.

2) AlertMe наблюдение за домом(газ, вода, количество человек в доме).
	Интерфес-гуи с скролящимся таб баром и автоматическое обновление состояния дома с сервера.
	Около 7 закладок для отображения той или иной информации.
		скрины IOS аналогичные android скринам:
			https://www.alertme.com/how-we-do-it/products-and-services/smart-heating/
			https://www.alertme.com/how-we-do-it/products-and-services/smart-energy/
			https://www.alertme.com/what-we-do/our-vision/

3) Обои несколько видов (Движение планет в космосе вокруг солнца, движение луны, летящие пузырьки разного размера...)
	Сделаные как на Обычном SurfaceView, так и на GLSurfaceView с исспользованием OpenGL(на java стороне)
	Отображение 2d, 3d текстур(Shader-ы)

4) Email-приложение для пожилых людей с упрощенным интерфейсом(урезание функционала 
	существующего email-приложения, часть UI убиралась. Были готовые исходники, их нужно было урезать.

5) Приложение для курьера(обработка заказов) 
	(android + серверная часть на Java. Получение сообщений(Android сервис+AlarmManager,
	запуск сервиса на старт устройства),
	кеш - который сделан по образу и подобию базы данных на сервере, HttpGet, HttpPos запросы
	AsynchroniusTask-и ThreadPoolExecutor-ы, самописный пул запросов с возможностью запуска синхронных
	и асинхронных задач, тесное взаимодествие с JSON-объектами, перерождение JSON объектов в SQL-запросы
	на серверной стороне с возможностью тестирования на android-клиенсткой стороне, при наличии трудностей
	доступа к серверу значительно ускоряет процесс разработки). Обработка результатов SQL запросов и кеширование
	в соответсвии с успешностью выполнения SQL запросов. Разработка ListView с ArrayAdapter с ViewHolder и setTag.
	Разработка кастомного контрола(тянущий скрол) на базе HorizontalPager со своими изменениями.
	Самописный переносчик текста. Самописный клас оберкта для работы с датой. Большая часть запросов в базу данных,
	которые осуществляются на сервер, история запросов на android стороне, работа с картой(google map api),
	фильтр заявок, раличные изменения статусов заявки. Отображение информации за день. Список а также краткий
	список заявок курьера на день. Частичная подгрузка данных опираясь на логирование изменений на сервере.
	Конфигурирование размера шрифта, настроек сервера(хост порт), логин/пароль курьера. Popup-меню в виде ListView.
	TabBar. Сортировка заявок-ListView при нажатии на элементы шапки(как в таблице)
	Эстимирование задач(по времени и сложности). Ведение проэкта в Jira (логирование времени,
	создание беклога, и резолв тасков) Подгрузка конфигурации приложения(Development\Production)
	в зависимости от настроичного файла в asserts. Автоматически Обновление приложение до следующей версии(при наличии таковой)
	Svn репозиторий. Сборка Java-сервера с помошью Jenkins(настройка jenkins) - тестовый сервер, релизный сервер,
	сборка Android обновления. Maven. 5 часовое накидывание Java-сервера SQL запросов на существующую базу данных,
	который в последствии раздает JSONObject-respons-ы для клиентов(Android-IOS).
	Сборка андроид версий клиентов и сервера полностью в пару кликов.

	Разработа архитектуры, исспользование шаблонов проэктирования, гибкой архитектуры, тестов(роболектрик + проэкт-мини для тестирования нагрузки сервера).

6) Приложение отображающие новости + при недоступности новостного сервера - отображение контента.(Исспользование Fragment-транзакций
	android-support-v4.jar). pulltorefresh
7) Подрузка UI по JSON-структуре. Приложение для отображения заказа билетов.
8) "Ты репортер" (добавить свой репортаж, запись фото, видео, аудио), посмотреть репортажи, выложеные другими людьми.
    https://play.google.com/store/apps/details?id=ru.rian.yourep&hl=ru
